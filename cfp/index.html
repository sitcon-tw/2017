SITCON 學生計算機年會，由對資訊抱持熱誠的學生們所組成的同名社群自發籌辦，希望能透過技術與知識的激盪，給予學生們一個用自身力量實踐夢想的舞台。今年主題「 Internet for Next Generation 新世代的網路挑戰」。身為學生開發者的我們，除了熟悉學術網路的使用外，是否更需要知悉網路治理等相關權益呢？從個資都集中在大企業等無法監督的雲端，到學術論文或是 MOOCs 等牽涉知識私有化、網路中立性等爭議，或是校園內討論最常使用的臉書老是存在變化多端的隱私權設定等等。面對種種的網路衝擊，使我們激盪出新的思考火花。

SITCON 2017 徵求切合主題「 Internet for Next Generation 新世代的網路挑戰」的稿件，當然一如往例，我們也很歡迎所有與資訊相關的稿件，期待有更棒的共鳴！

投稿主題範例

「新世代的網路挑戰」是 2017 的主題，投稿不一定要以此為主題，但若有與主題關聯之投稿，能夠與之相呼應，我們非常歡迎！

我們特別期待以學生為主體的投稿，可以是從學生角度出發的經驗分享、技術分享，或是探討學生相關議題等等，不過廣義來說，只要是與資訊技術、電腦相關的講題，都歡迎投稿！

以下的投稿範例可以提供一些靈感，或是您可以參與投稿者小聚，亦可前往 SITCON 前幾屆的網站（2013、2014、2015、2016）查看過去的議程。

Code of Conduct

SITCON 非常歡迎女性、跨性別與多元背景的投稿者。為了讓大家都能愉快的參加 SITCON，我們鼓勵所有參與者閱讀年會的 行為守則（Code of Conduct），共同創造一個友善的環境。

議程種類

Talk / Short Talk

Talk 為 30 分鐘的議程，加上 10 分鐘的問答時間；Short Talk 為 10 分鐘的議程，問答時間由講者自行安排。兩者僅有時間上的差別。

Lightning Talk

大會當天早上開放報名，每位講者有 3 分鐘，3 分鐘一到可是會被趕下臺的！

Unconference

Unconference 是指「非正式會議」，不同於由大會排定的議程，Unconference 是由所有參與的會眾共同排定的。在會眾踴躍的投遞完想要分享的主題之後，透過票選，會眾自行決定想聽的講題。因此每個人都有可能是講師，也沒有人可以預料 Unconference 會出現什麼樣千奇百怪的新鮮議題！

每位講者有 20 分鐘，於交誼廳演講。

投稿方式

稿件均經議程組形式審查，交予該領域之審稿委員匿名審稿；審稿委員由業界人士、學界人士及 SITCON 議程組員組成，依照下列評分方式評分，錄取高分者，稿件一律以速審為原則。

格式（Talk 與 Short Talk 的投稿格式相同）

  • 題目 Title：必填，20 全形字以下。
  • 摘要 Abstract：必填，含標點符號 150 ~ 250 全形字，安排於講者海報及網站。
  • 詳細說明 Outline：必填，不限字數，請說明這個議程的大綱、目的及目標聽眾，及任何你認爲重要的相關細節，以利審稿委員更瞭解你的作品。我們強烈建議您詳細填寫此項，以增加入選機率。
  • 議程介紹 Pitch：選填，請說明爲什麼你覺得這個議程適合這個活動，以及爲什麼你想與大家分享。
  • 個人介紹 Bio：必填，含標點符號 50 ~ 100 全形字，安排於講者海報及網站。
  • 投稿類型：請寫明所欲投稿者為 Talk 或 Short Talk。

個人資料（請至 Profile 中填寫，以下只列出必填項目）

  • 電子郵件 Email：能用於大會聯絡的電子信箱。
  • 暱稱/名字 Name：公開於講者海報及網站的稱呼。
  • 個人頭像 Photo：可以是任何不違反 CoC 之圖片，請提供 1000 x 1000 像素以上之 JPG/PNG 圖檔。
  • 電話號碼 Phone：用於大會聯絡、確認資訊用的電話號碼。

SITCON 2017 採用投稿系統接收投稿，需使用 Github OAuth 登入新增投稿，若沒有 Github 帳號請先至 Github 註冊。（同一帳號僅能為一人使用，一人能投多份稿件）

一個半形字元計為 0.5 個全形字元，一切投稿相關規定事項以本文件為準。

評分方式

審稿委員主要針對以下項目進行評分

  • 稿件內容：包含的知識、經驗、想法、觀點等，還有是否有趣、酷炫
  • 表達能力：提供的資料是否有條理、文句暢通，提供的資料（尤其是稿件的詳細說明）的完整度，我們偏好完整的資料，讓審稿委員更清楚了解演講細節
  • 適合聽眾：稿件是否適合 SITCON 大多數的會眾，若經我們評估比較小眾的議程可能會被安排至較小的演講廳

投稿注意事項

  • SITCON 2017 有 3 軌議程同時進行（不計 Unconference 及 Lightning talk）。
  • Lightning talk 將於所有議程結束後，閉幕前於國際會議廳舉行。
  • 上述評分方式不考慮稿件內容的技術難度，我們仍以稿件是否能夠吸引人、高品質、適合聽眾（即上述評分方式）為考量。
  • 為鼓勵投稿，大會提供投稿人一組邀請碼 （一人一組，非一稿一組），可優先報名參與本年會，獲選講者請使用講者身分入場，並可將邀請碼給他人使用。
  • 在截稿之前，投稿者可以隨時使用投稿系統修改已投的稿件。
  • SITCON 議程組得經作者同意，轉換 Talk 或 Short Talk 稿件之投稿類型。
  • 無論有無入選，仍然歡迎於年會當天報名 Lightning Talk 或 Unconference！
  • 凡稿件經接受者，年會將頒予感謝狀以表彰投稿人之熱情付出與貢獻。
  • 講者海報僅於年會當日於年會場地張貼，可能會將議程簡介放置官網簡介與會前發行的電子報中。

若有任何問題或需投稿建議請 Email 至 SITCON 議程組信箱 <session (at) sitcon.org>。

錄影及直播

所有議程皆會錄影及線上直播，若不希望錄影可以與議程組聯絡。講者可以選擇將錄影以創用CC或其他授權方式釋出。

為了配合線上直播的需求，本屆 SITCON 提供入選講者試講及彩排的機會。試講除了讓大會工作人員先行對議程內容、演講習慣有初步了解外，也可以讓講者熟悉現場流程、找出能讓簡報更生動精采的模式。彩排則讓講者熟悉現場設備環境、攝影機位置，並且測試 Live Demo 情境。希望講者能夠共同參與試講及彩排，一起使年會直播品質更臻完美。

試講

試講讓講者在演講前有練習的機會，比照年會規格講完整場演講，並有議程組人員提供建議（如簡報、台風等），講者可以自由參加，自行選擇最接近的場次，入選後將由議程組與您協調精確時間。

所有場次將依報名情況舉辦，並提供投影機、音響、簡報筆、倒數計時器等設備。預計舉辦北部、中部、南部、東部場等，時間約為 2 月份，地點另行通知。

補助：依照客運價格標準補助講者至最近試講場地的全額車馬費
